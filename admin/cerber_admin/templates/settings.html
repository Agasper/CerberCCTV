{# ------------------------------------------------------------ Камера #}

RTSP — протокол, по которому IP-камеры отдают видео. Агент подключается к камере по локальной сети и читает поток напрямую, ничего не устанавливая на камеру.

Адрес собирается так:

rtsp://логин:пароль@IP-камеры:554/путь

  • логин/пароль — учётка от веб-интерфейса камеры;
  • 554 — стандартный порт RTSP (обычно менять не нужно);
  • путь зависит от производителя. Hikvision: /Streaming/Channels/101 — основной поток, /Streaming/Channels/102 — субпоток. Dahua: /cam/realmonitor?channel=1&subtype=0 (subtype=1 — субпоток).

Спецсимволы в пароле нужно закодировать: @ → %40, : → %3A, / → %2F, % → %25. Например, пароль a@b@ в адресе выглядит как a%40b%40.

Субпоток включается в настройках камеры (у Hikvision: Video/Audio → Stream Type → Sub-stream); рекомендуемые параметры — 640×480 @ 5–10 к/с, H.264.

Проверить адрес можно в VLC: Медиа → Открыть URL — если поток играет там, заработает и здесь.

Обведите мышью области, где нужно искать движение: клик добавляет точку, «Замкнуть зону» завершает полигон. Зон может быть несколько. Без зон анализируется весь кадр. Порог «мин. площадь движения» считается от площади зон.

{# --------------------------------------------------------- Хранилище #}

DigitalOcean Spaces
  • Endpoint: https://fra1.digitaloceanspaces.com (замените fra1 на регион вашего Space)
  • Регион: fra1; Path-style: выключен
  • Ключи: панель DO → API → Spaces Keys
Amazon S3
  • Endpoint: пусто; Регион: регион бакета, например eu-central-1; Path-style: выключен
  • Ключи: IAM-пользователь с правами на бакет (GetObject, PutObject, DeleteObject, ListBucket, AbortMultipartUpload)
Google Cloud Storage
  • Endpoint: https://storage.googleapis.com; Регион: регион бакета; Path-style: включён
  • Ключи: HMAC-ключи в Cloud Storage → Settings → Interoperability
Backblaze B2
  • Endpoint: https://s3.eu-central-003.backblazeb2.com (точный адрес — в свойствах бакета)
  • Регион: из endpoint, например eu-central-003; Path-style: выключен
  • Ключи: App Keys (мастер-ключ не работает с S3 API)
Bunny
  • У Bunny Edge Storage свой API, S3-совместимый режим появился недавно и заявлен как бета — endpoint и ключи смотрите в панели Bunny (Storage → FTP & API Access). С Cerber не проверялось; если не заведётся — надёжные варианты выше.
